# 需要确认的点

根据目前了解到的情况，需要明确以下几点：

## 1. 主营业务确认

**（对视觉设计会有影响）**

- 目前了解到的是古董的出售，这个是否正确？
- 如果不正确：
  - 出售的商品是什么类型？
  - 是单一品类还是多品类？

## 2. 付款方式

- 目前确认到的是只需要银行转账，这个是否属实？
- 如果不正确：
  - 会包含哪些收款方式？

## 3. 网站定位

这个网站的定位是什么？

- [ ] 纯粹的购物网站
- [ ] 购物网站 + 部分内容展示（主营商品的相关资讯等）
- [ ] 公司网站 + 内容展示 + 购物网站

## 4. 设计风格确认

- 参考网站的设计风格是否也需要参考？
- 如果不是：
  - 希望网站呈现什么样的风格？（例如：专业、高端、简洁、传统、现代等）
  - 有没有其他喜欢的网站可以参考？

## 5. 部署方式的确认

> 💡 **详细技术方案请参考：**[方案](方案.md)

**共通说明：**

- 项目代码将托管在 Git 仓库中（如 GitHub）
- 项目完成后可将仓库权限转移给您
- 未来需要时可以让任何开发者进入继续开发

### 方案一：服务器部署

**（云服务器或自建服务器）**

- 需要自行准备服务器（或购买云服务器）
- 将代码部署到服务器上运行
- 可设置自动部署（代码更新后自动上线）
- 适合需要完全控制服务器环境的情况

### 方案二：Cloudflare 全托管服务

**（成本低且易维护）**

- 无需购买服务器，托管在 Cloudflare 平台
- 免费额度对中小型网站足够使用
- 自动扩展，无需担心流量问题

**免费提供以下服务：**

- **Workers**：网站本体，免费
- **D1**：数据库，记录商品、用户信息等，5GB 限额，超出才开始收费
- **R2**：文件存储，存储图片等资源，10GB 限额，超出才开始收费
- **KV**：辅助本体运行，免费

- **网站结构以及内容设计**：根据[参考网站网站结构](参考网站网站结构.md)已作成初版[新网站设计结构](新网站结构.md)，详细设计得具体确认

- 是否已有域名？
- 如果没有，希望的域名是什么？ 需购买，可代为操作并提供指导后续维护指导等

## 8. 项目时间和预算

- 期望的上线时间？
- 对于项目预算，您这边的心理预期大概是多少？
  - 这样可以根据预算调整功能范围和实现方式，确保在合理范围内完成
- 对于项目，会提供免费的维护时间，但是需要详细商谈范围，以及追加功能所需花费
- 对网站的SEO和性能要求多高，要求越高费用越高
